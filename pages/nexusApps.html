CGM2.i

Updating the Conventional Gait Model for the modern world

Nexus Application Environment

Figure below despict the pyGM2 nexus application environment

Users must mainly take note of the articulation between Vicon Nexus and the folder programData/pyCGM2.

Folder programData/pyCGM2

This folder1 is installed after first importation of pyCGM2. For each Conventional Gait Model, this folder collects :

  • ready-to-import Nexus pipeline (file .pipeline)
  • global settings (file .settings)

The role of Global settings si to set commonly-use settings. As an example, for the CGM1.0 ( i.e PiG clone), global settings declares by default :

  • marker diameter to 14mm
  • enable both flat foot options !!
  • project moment into the distal segment
User is free to alter default values of global settings if they don't match his routine.

Nexus application

From the ressource/pipeline panel of Vicon Nexus, you can import ready-to-use pipeline file (see video).

In the above figure the Calibration pipeline of the CGM1 was imported/ Vicon Nexus displays the serie of operations, including pyCGM2 operations and native Nexus operations.

By selecting a python operation, the panel properties displays

  • the location the python script file
  • script arguments. Role of these arguments is to overload global settings and provide options.

    User must refer to each CGM section or the API Documentation to know possible script arguments of pyCGM2 operations.

  • a check box Use nexus python.

    Must be unchecked.

  1. programData folder might be set to hide in your microsoft window environment. Change this option in your file explorer to display this folder ↩
